## Doel

Eenrichting-sync van runyo-backend naar Google Sheets voor users die hun schema ook op desktop willen onderhouden. Settings-toggle om aan te zetten, on-demand sync-button + periodic cron. Sheet-mapping op een door user opgegeven `sheet_id` + `tab_name`. Geen reverse-sync in v4 — Sheet is een mirror, niet een tweede bron-van-waarheid.

## Acceptance

- User kan Sheets-sync aanzetten in instellingen door een sheet-URL te plakken (`sheet_id` afgeleid + `tab_name` opgevraagd)
- Sync-knop schrijft alle activities + races naar de gekozen Sheet/tab
- Activities verschijnen daar in het Activity-formaat (zelfde kolomvolgorde als de huidige PWA-Sheet, voor leesbaarheid)
- Periodic sync via Railway cron (1x per uur of bij elke activity-mutatie — kies eenvoudigste optie tijdens implementatie)
- Sync-fouten (Sheets-API 401/403/404, malformed-tab) worden in settings-UI getoond met laatste-poging-timestamp
- Tests: `sync.ts` unit tests — `formatActivitiesForSheet(activities)` returnt correct 2D-array (≥2 cases), `applySyncDiff` test op idempotency (≥1)

## Test-strategie (red-green-refactor)

1. **Red** — schrijf `runyo-auth/tests/sheets-sync.test.ts` met `formatActivitiesForSheet([{datum: '2026-06-01', type: 'run', km: 10, ...}])` → `[[ 'datum', 'type', ... ], [ '2026-06-01', 'run', ..., 10 ]]`. Faalt want functie bestaat niet.
2. **Green** — implementeer `sync.ts` (formatter + Sheets-API-writer) + endpoint `POST /api/sheets-sync/run` + de cron-trigger. Settings-UI: nieuw scherm `SettingsSheetsSync.tsx`.
3. **Refactor** — als formatter en parser-bij-import-2.2 dezelfde kolom-mapping delen, extract een `activityColumnMap` constante.

## Tech-hints

- Hergebruik de bestaande Google-OAuth-scopes — user is al ingelogd met Google, `spreadsheets.write` scope toevoegen aan auth-request (PRD §5.1 Google OAuth).
- Sheet schrijven via `https://sheets.googleapis.com/v4/spreadsheets/{id}/values/{tab}!A1:append` (of `:update` als we de hele tab vervangen — eenvoudiger).
- Cron: hergebruik de scheduler die voor push-notifs in 2.5 is opgezet. Eén cron-process, meerdere job-types.
- Valkuil: users die niet met Google zijn ingelogd (Apple/email — zie 2.3) hebben geen Sheets-scope. Settings-UI moet die case detecteren en uitleggen ("Sheets-sync vereist Google sign-in").
