# 2.4 — Edits-flow: drag-reschedule + verwijder + rust + feedback

**Fase:** 2 (Core features)
**Type:** AFK

**Depends on:** 2.1

## Doel

Alle muteer-flows die op de bestaande backend leunen activeren: drag-en-drop tussen dagen in WeekScreen (= `PATCH ... { datum }`), verwijderen via swipe of long-press met undo-toast, markeer-als-rust (= `PATCH ... { type: 'rest' }`), en feedback-submit op TodayScreen (rating + tekst). De backend hoeft niets nieuws — dit zijn vier UI-flows bovenop bestaande endpoints.

## Acceptance

- Drag-en-drop tussen dagen werkt op iOS én Android én web (touch + muis)
- Drag is optimistisch: UI verschuift direct, rollback bij PATCH-error met toast
- Delete is reversible binnen 5 seconden via undo-toast (timeout daarna echt DELETE)
- Markeer-als-rust beschikbaar in DayDetailModal én als swipe-actie in WeekScreen
- Feedback-submit op TodayScreen: 5-emoji rating + optionele tekstinput → slaat op via `PATCH /activities/:id` met `rating` + `feedback`
- Bij re-open van dezelfde dag is de gegeven feedback en rating zichtbaar
- Tests: unit tests op `activityEdit.ts` validatie (≥3 — ongeldige datum, type-coercion, ongeldige rating-range)
- Tests: component-test op feedback-submit-flow (RNTL: input → submit → API-call gemockt → success-state)

## Test-strategie (red-green-refactor)

1. **Red** — schrijf `src/services/activityEdit.test.ts` met `validateRating(6)` → ValidationError en `coerceToRest({ km: 10, type: 'run' })` → `{ km: null, type: 'rest', ... }`. Beide falen.
2. **Green** — implementeer in `activityEdit.ts` (bestaand uit 1.3) de mutatie-helpers + extra validatie. Sluit aan op `PATCH`-call. Drag-handler in WeekScreen roept een nieuwe `reschedule(activityId, newDate)` aan die optimistisch is.
3. **Refactor** — als de drie mutatie-flows (reschedule/delete/rest) dezelfde optimistic-update-shape hebben, extract een `optimisticUpdate(queryKey, mutator)` helper bovenop TanStack Query. Niet eerder dan na de derde herhaling.

## Tech-hints

- Drag-en-drop: `react-native-reanimated` + `react-native-gesture-handler` op native; op web werkt dat ook (Reanimated 3 ondersteunt web). Geen aparte web-implementatie nodig.
- Undo-toast: gebruik een lichte custom Toast-component bovenop bestaande primitives; 5s timer met `setTimeout` is voldoende. Bij undo: stuur de cancel vóór de DELETE-call vertrekt (queue de DELETE met delay i.p.v. direct).
- Feedback-veld: rating is `int 1-5`, feedback is `text` max 1000 chars. Backend valideert opnieuw — frontend-validatie is UX, niet security.
- Valkuil: optimistic drag-update mag de TanStack Query-cache niet inconsistent maken als twee drags snel achter elkaar gebeuren. Gebruik `onMutate` + `onError` rollback-pattern uit TanStack docs.
