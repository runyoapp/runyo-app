# 3.1 — Web build + directe cutover

**Fase:** 3 (Launch)
**Type:** HITL (deploy + DNS-config)
**Lagen:** build-tooling + hosting
**Depends on:** alle fase-2 slices in scope voor v4.0
**Blocks:** 3.4
**Status:** pending

## Doel

`react-native-web` + `react-dom` + `@expo/metro-runtime` installeren, responsive layout verifiëren, `npx expo export --platform web` uitvoeren, 1-2 weken dogfooden, daarna `dist/` deployen naar `app.runyo.app` ter vervanging van de huidige PWA. Oude PWA-code wordt geärchiveerd als `legacy-pwa` branch. Effectieve launch van runyo v4 op web.

## Acceptance

- `npx expo export --platform web` slaagt zonder errors of warnings die op runtime-issues wijzen
- Responsive layout: bottom-tabs op viewport <768px, sidebar of 2-column op ≥768px
- Lokale of preview-deploy van de bundle is 1-2 weken dagelijks gebruikt door user zonder de oude PWA te openen (= dogfood-milestone uit PRD §8)
- `app.runyo.app` toont nieuwe bundle; oude PWA niet meer bereikbaar op root
- `legacy-pwa` branch bestaat in `runyoapp/runyo-app`, vanaf de laatste pre-cutover `main`-commit (gepushed naar origin)
- `app.json` bumped naar `4.0.0` op moment van cutover
- `CLAUDE.md` (in `runyoapp/claude`) sectie "XApp / runyo-app (frontend)" geüpdatet naar Expo/RN/TypeScript
- Rollback-pad getest: revert van de Pages/Cloudflare-deploy naar `legacy-pwa` werkt binnen 5 min

## Verificatie

Geen geautomatiseerde tests — deze ticket is operationeel. Handmatig:
1. Open `app.runyo.app` op iPhone-Safari, Android-Chrome, desktop-Chrome, desktop-Safari en verifieer dat de nieuwe bundle laadt en de golden path werkt.
2. Test rollback in een staging-env: revert deploy → oude PWA komt terug → revert ongedaan → nieuwe bundle komt terug. Notitie van wall-clock-tijd in PR.
3. Check DNS-record en HTTPS-cert op `app.runyo.app` — geen mixed-content of stale-cache issues.

## Werkverdeling — agent vs. mens

**Agent:**
- Installeert web-deps via `npx expo install react-native-web react-dom @expo/metro-runtime`
- Past `app.json` aan voor web (`web.bundler: "metro"`) en zet `app.json` versie op `4.0.0`
- Voert responsive layout-fix uit voor screens die op desktop breken (sidebar, max-width wrappers)
- Doet `expo export --platform web` en levert de `dist/` map
- Update `CLAUDE.md` frontend-sectie naar Expo/RN
- Schrijft een rollback-runbook (commands + waar revert-knop in Pages/Cloudflare zit)

**Mens:**
- Doet de 1-2 weken dogfood (alleen jij kan beoordelen of het "goed genoeg voelt")
- Klikt in GitHub Pages / Cloudflare Pages de nieuwe deploy live
- Configureert DNS-record voor `app.runyo.app` als die nog niet pointed (zie open actie in `CLAUDE.md`)
- Voert de rollback uit als dat nodig blijkt
- Maakt de `legacy-pwa` branch en pusht hem
