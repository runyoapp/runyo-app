# 2.3 — Auth-breedte: Apple Sign-In + email/wachtwoord

**Fase:** 2 (Core features)
**Type:** AFK (subset HITL: Apple Developer Account configuratie)
**Lagen:** db + backend + frontend
**Depends on:** 1.2
**Blocks:** —
**Status:** pending

## Doel

Uitbreiden van Google-only auth naar drie methodes: Google (bestaand), Apple Sign-In (nieuw), email/wachtwoord (nieuw). Backend krijgt `users.password_hash` (argon2) en `users.apple_sub`, plus de endpoints `POST /auth/email-signup`, `POST /auth/email-signin`, `POST /auth/apple`. Frontend login-scherm krijgt drie keuze-knoppen. Account-linking gebeurt op email-match — dezelfde echte persoon krijgt één `user_id` ongeacht via welke methode ze inloggen.

## Acceptance

- Alle drie auth-methodes leiden tot dezelfde `user_id` voor dezelfde email-adres (linking-rule)
- `POST /auth/email-signup` valideert: min 10 chars, niet in een korte common-passwords-lijst (top-100 hardcoded is ok)
- `POST /auth/email-signin` retourneert 401 bij verkeerd wachtwoord (geen leak of email bestaat)
- `POST /auth/apple` verifieert Apple-token server-side (key-fetch + signature-check) — geen client-trust
- Frontend toont 3 knoppen op het login-scherm, zelfde visuele gewicht
- Tests: argon2 hash-roundtrip (≥1), email-conflict-resolution bij Apple-na-Google linking (≥1), Apple-token verify met mock-key (≥1), password-rules guard (≥1)

## Test-strategie (red-green-refactor)

1. **Red** — schrijf `runyo-auth/tests/auth.test.ts`: signup user-A met email+pw, daarna sign-in same email/correct-pw → 200 + zelfde `user_id`; sign-in same email/wrong-pw → 401. Verifieer dat een tweede sign-in via Apple met identiek email-claim de bestaande user-id teruggeeft. Faalt want endpoints bestaan niet.
2. **Green** — implementeer Drizzle-migratie (kolommen toevoegen aan `users`), argon2-hashing (`@node-rs/argon2` of `argon2` package), drie handlers, en de Apple-token-verify (fetch `https://appleid.apple.com/auth/keys`, JWT signature check).
3. **Refactor** — extract een `findOrCreateUserByEmail(email, { google_sub?, apple_sub? })` helper als alle drie de handlers dat patroon herhalen.

## Tech-hints

- Argon2 boven bcrypt (PRD §5.1) — moderne defaults, snellere reviews door auth-experts.
- Apple Sign-In op web werkt via `@invertase/react-native-apple-authentication` (native) en hun web-SDK (Expo web). Backend-verify is identiek.
- HITL-deel: Apple Developer-account configuratie (Service ID, Key, Return URL `https://app.runyo.app/auth/apple/callback`) — agent maakt een checklist + screenshots-template; user voert in console uit en plakt de Key-ID + Team-ID + private key terug in Railway env-vars.
- Valkuil: email-linking met case-insensitivity. Normaliseer alle emails naar lowercase voor lookup én opslag. Anders krijgt `Foo@x.com` en `foo@x.com` twee accounts.
- Common-passwords-lijst hardcoded klein houden (top 100, ~2KB) — geen externe call op signup.
