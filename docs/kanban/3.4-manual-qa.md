# 3.4 — Manuele QA-pass

**Fase:** 3 (Launch)
**Type:** HITL
**Lagen:** alle
**Depends on:** alle slices in scope voor v4.0
**Blocks:** — (laatste ticket vóór v4.0-completeness)
**Status:** pending

## Doel

Volledige golden-path en edge-case-checklist doorlopen op vier platforms (iOS-device, Android-device, mobile-Safari, desktop-Chrome). Per finding een bug-ticket aanmaken — gefixed binnen v4.0 of expliciet geaccepteerd-voor-v4.1. Eindigt met een go/no-go-beslissing voor het cutover-moment van 3.1.

## Acceptance

- Golden path werkt op alle 4 platforms (iOS-native, Android-native, mobile-web Safari, desktop-web Chrome):
  - sign-in → import → vandaag-view → week-view edit → training-detail edit → feedback geven → push-notif ontvangen
- Edge-case-list afgevinkt:
  - no-internet (offline read-only werkt, edit toont duidelijke melding)
  - expired Google-token (refresh werkt, anders re-auth-prompt)
  - missing notification-permission (UI legt uit, biedt knop naar settings)
  - Telegram niet gekoppeld (geen errors, push-fallback werkt)
  - tweede user inloggen op zelfde device (data-leak-check)
- Alle gevonden bugs zijn tickets in `docs/kanban/` (status `pending`, `in_progress`, `completed` of `deferred-v4.1`)
- Geen P0/P1 bugs open op het moment dat 3.1 (web-cutover) groen licht krijgt
- Een korte QA-rapportage in deze ticket (markdown-comment of nieuwe `3.4-report.md`) met: platform-matrix, finding-list, go/no-go-aanbeveling

## Verificatie

Volledig handmatig. Geen automated tests in deze ticket.

## Werkverdeling — agent vs. mens

**Agent:**
- Genereert de QA-checklist als markdown-template (golden-path + edge-cases, één rij per platform-combinatie)
- Maakt voor elke gevonden bug een nieuw ticket-file in `docs/kanban/` volgens hetzelfde template (id-conventie: `qa-NNN-<short-name>.md`)
- Schrijft bug-fixes voor de bugs die in scope vallen voor v4.0 (de fix zelf is een eigen ticket maar de agent voert hem uit)

**Mens:**
- Voert de daadwerkelijke QA-pass uit op de vier platforms — alleen jij hebt fysieke devices en account-toegang
- Beoordeelt per bug: in-scope-v4.0 (fix) of deferred-v4.1 (accepteer)
- Drukt de go/no-go-beslissing voor 3.1
- Tekent af op de uiteindelijke QA-rapportage
