# 3.2 — Native builds + store submission

**Fase:** 3 (Launch)
**Type:** HITL (Apple Developer + Google Play Console accounts, screenshots, descriptions)
**Lagen:** build + store-tooling
**Depends on:** alle fase-2 slices in scope voor v4.0
**Blocks:** 3.4 (gedeeltelijk — manuele QA op native kan parallel)
**Status:** pending

## Doel

`eas build --profile production` voor iOS en Android draaien, TestFlight-upload + Play Console internal-track, store-listings invullen (screenshots, descriptions, privacy-policy-link, leeftijdslimiet), submit voor review. Native-launch loopt parallel met of na de web-cutover (3.1) — store-approval is geen blocker voor web.

## Acceptance

- Beide EAS-builds slagen zonder errors voor `production`-profile
- iOS-build installeerbaar via TestFlight op test-device(s)
- Android-build installeerbaar via Play Console internal track op test-device(s)
- Beide app-store-listings ingevuld: app-icon, screenshots (mobile + tablet), description (NL + EN), privacy-policy-link naar `runyo.app/privacy`, leeftijdslimiet ingesteld, categorie "Health & Fitness"
- Apple Push Certificate uit 2.5 is gekoppeld aan de iOS-build (push werkt op TestFlight-install, niet alleen simulator)
- Beide apps submitted voor review
- Geen kritieke bugs gedurende 2 weken eigen gebruik op de native builds (PRD §8 milestone)

## Verificatie

Geen unit-tests in deze ticket — eindpunt is een externe approval. Handmatig:
1. Install van TestFlight + internal-track op fysieke test-device(s).
2. Golden path doorlopen (zie 3.4 voor de checklist) — direct na install + na 24u (= push-notif aankomst).
3. Bij review-rejection: lees feedback, fix in een nieuw mini-ticket, resubmit.

## Werkverdeling — agent vs. mens

**Agent:**
- Schrijft/update `eas.json` met `production` profile voor iOS en Android
- Genereert app-icon-set en splash-screen op de juiste resoluties als ze niet compleet zijn
- Schrijft drafts van de store-descriptions (NL + EN) op basis van PRD §1-2
- Genereert lijst van vereiste screenshots (welke screens, welke resoluties per device-class)
- Documenteert de submit-checklist (privacy-policy, age-rating, data-safety-form voor Play)
- Genereert een changelog/release-notes-template

**Mens:**
- Apple Developer Account aanmaken/onderhouden (jaarlijks $99)
- Google Play Console-account aanmaken (eenmalig $25)
- Push Certificate (.p12) genereren in Apple Developer Console en aan EAS koppelen
- Screenshots maken op echte devices of via Simulator/Emulator
- Submit-knop drukken in App Store Connect en Play Console
- Reageert op review-feedback (vereist account-toegang)
- Verifieert dat 2-weken-eigen-gebruik milestone is gehaald voordat publieke release
