# 2.2 — Importer schrijft naar backend i.p.v. Sheets

**Fase:** 2 (Core features)
**Type:** AFK
**Lagen:** backend + frontend
**Depends on:** 2.1, 1.3
**Blocks:** —
**Status:** pending

## Doel

De import-pipeline eindigt niet meer in een Google Sheet maar in runyo's backend. `/ai/import` geeft parsed schema terug; frontend roept `POST /api/schemas` + bulk `POST /api/schemas/:id/activities`. ImportFlow eindigt op TodayScreen met populated backend-data. Excel/Sheets-format en URL-format worden toegevoegd aan de importer (PDF en foto bestaan al).

## Acceptance

- Alle 4 import-formats werken: PDF, foto, Excel/Google Sheet (file of URL), webpagina-URL
- `/ai/import` retourneert `{ schema: { source_type, source_metadata }, activities: [...], races: [...] }` als pure data, geen Sheets-write
- Frontend ImportFlow: na confirm doet `POST /api/schemas` → krijgt `schemaId` → `POST /api/schemas/:id/activities` met de bulk-payload (of N keer single-`POST` als bulk-endpoint later komt)
- Na geslaagde import landt user op TodayScreen met >0 activities zichtbaar
- Fouten in de AI-respons (malformed JSON, lege activities-array) tonen een duidelijke error-state in ImportFlow, geen silent fail
- Tests: import-service unit tests per format (parse-only, gemockte AI-response) — minimaal 4 tests
- Tests: 1 integration test op de happy-path (mock `/ai/import`, mock `POST /activities`, verifieer dat `dataStore.activities` populated is)

## Test-strategie (red-green-refactor)

1. **Red** — schrijf `src/services/import.test.ts` cases voor de twee nieuwe formats: `parseExcel(mockBuffer)` en `parseUrl(mockHtml)` retourneren genormaliseerde activity-arrays. Beide falen want nieuwe format-handlers bestaan niet.
2. **Green** — voeg de twee format-handlers toe in `import.ts` (uit 1.3) + bijbehorende prompt-templates voor `/ai/import`. Pas ImportFlow aan om aan het einde de backend-calls te doen i.p.v. een Sheet aan te maken.
3. **Refactor** — als de vier format-handlers een gedeelde shape hebben (input → genormaliseerde rows), extract een `ImportFormat` interface; gebruik die in ImportFlow voor het format-keuze-scherm.

## Tech-hints

- `/ai/import` blijft in `runyo-auth` als proxy naar Anthropic; alleen de response-shape verandert (geen Sheet-write meer, return parsed data).
- Excel/Sheets parsing: client-side via `xlsx` of laat AI het rauwe CSV-dump verwerken. Kies AI-pad — minder client-bundle, consistent met PDF/foto.
- URL-import: backend fetcht de pagina (CORS-vrij) en geeft de HTML/text aan de AI. Voorzie een 10s-timeout.
- Bulk-insert: voor v4.0 mag het N single-POSTs zijn (eenvoudig); voeg `POST /api/schemas/:id/activities/bulk` toe alleen als de N>~50-case echt traag voelt in dogfood.
- Valkuil: `source_metadata` in `schemas` tabel moet genoeg info bewaren om bij re-import te detecteren dat het dezelfde bron is (bv. filename + hash, of URL). Niet voor v4.0 noodzakelijk maar laat het schema het wel toe.
