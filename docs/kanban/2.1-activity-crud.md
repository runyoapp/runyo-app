# 2.1 — Activity CRUD end-to-end

**Fase:** 2 (Core features)
**Type:** AFK
**Lagen:** db + backend + frontend
**Depends on:** 1.2
**Blocks:** 2.2, 2.4, 2.6, 3.3
**Status:** pending

## Doel

`activities` + `races` tabellen aanmaken en de vier CRUD-endpoints (`GET /api/schemas/:id/activities`, `POST`, `PATCH`, `DELETE`) opleveren. Frontend `services/activities.ts` vervangt de huidige Sheets-calls, `dataStore.activities` wordt gevoed door backend. TodayScreen en DayDetailModal lezen/schrijven via backend i.p.v. Sheets.

## Acceptance

- Alle 4 endpoints werken met Google-token-auth (hergebruik middleware uit 1.2) en strikte user-isolation (geen activity-IDs van andere users te lezen/muteren)
- `POST /activities` valideert `datum` (ISO), `type`, en optioneel `titel`, `detail`, `km`
- `PATCH` accepteert partial body; ongeldige velden → 400
- `DELETE` is idempotent (404 als id niet bestaat of niet van user)
- TodayScreen toont activities uit backend (geen Sheets-fetch meer in dit pad)
- DayDetailModal kan een activity bewerken via backend en de wijziging is bij re-open zichtbaar
- Tests: backend integration tests voor CRUD round-trip + user-isolation (≥4 cases)
- Tests: service unit tests op `activities.ts` (1 per endpoint, happy + 1 sad)

## Test-strategie (red-green-refactor)

1. **Red** — schrijf `runyo-auth/tests/activities.test.ts` met round-trip: user-A maakt schema + activity, user-B `GET`'t en krijgt lege array; user-B `PATCH`'t het id van A en krijgt 404. Faalt want endpoints bestaan niet.
2. **Green** — implementeer Drizzle-schema's voor `activities` + `races`, foreign keys op `schemas.id`, en de vier handlers. Hergebruik `requireAuth` uit 1.2.
3. **Refactor** — zodra alle vier handlers er staan, extract een `loadOwnedSchema(schemaId, userId)` helper als drie ervan dezelfde guard hebben. Niet eerder.

## Tech-hints

- `races` tabel hier al meenemen — PRD §5.2 noemt ze; race-UI-flows komen later maar de tabel kost niets nu.
- `activities.type` is enum-string (`run`, `interval`, `long`, `rest`, ...). Houd in Drizzle als `text` met TypeScript-union, geen Postgres-enum — eenvoudiger te migreren.
- Backend: `runyo-auth/src/routes/activities.ts` nieuw, mounted op `/api/schemas/:schemaId/activities`.
- Frontend: `src/services/activities.ts` vervangt de Sheets-aanroepen in `src/services/sheets.ts` voor dit pad; `dataStore` selector blijft hetzelfde shape zodat screens niet hoeven aan te passen.
- Valkuil: TanStack Query cache-invalidation bij PATCH/DELETE — invalideer de `['activities', schemaId]` key, niet alleen het individuele item, anders blijft Week-view stale.
