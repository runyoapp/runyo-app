## Doel

Native push-notificaties verzenden via `expo-notifications` (Expo Push Service). Backend krijgt `push_tokens` + `notification_prefs` tabellen en een scheduled task die per user op het juiste lokale tijdstip een push verstuurt met de training-van-morgen-info. Frontend: permission-flow, token-registratie bij sign-in, instellingen-scherm voor tijd + welke notif-types aan staan.

## Acceptance

- Test-device(s) ontvangen push op het ingestelde lokale tijdstip
- Lokale tijdzone wordt correct gebruikt (user's tz wordt opgeslagen bij token-register)
- Permission-denied: app toont uitleg + knop om naar device-instellingen te gaan, geen silent-fail
- Multi-device per user: registreren van een tweede device voegt token toe, oude token blijft (logout verwijdert het token)
- Instellingen-scherm: tijd-picker, type-toggles (vandaag-reminder, morgen-preview), opslaan triggert `POST /api/notification-prefs`
- Tests: cron-job-logic unit test (welke users moeten op tijdstip X een push krijgen, gegeven mock-tz-data) — minimaal 3 cases inclusief tz-edge (user in UTC+13)
- Tests: endpoint integration test op `POST /api/push-tokens` (happy + duplicate-token + 401)

## Test-strategie (red-green-refactor)

1. **Red** — schrijf `runyo-auth/tests/push-scheduler.test.ts` met een mock-clock op 20:00 lokaal voor 2 users (NL en NZ). Verwacht: `selectUsersToNotify(now)` retourneert de NL-user, niet de NZ-user. Faalt want functie bestaat niet.
2. **Green** — implementeer `push-tokens` + `notification_prefs` Drizzle-schema's, de scheduler-functie (pure functie die `now: Date` en een users-array neemt) en de Railway cron-trigger die elke minuut `selectUsersToNotify(new Date())` aanroept en pushes verstuurt via Expo Push API.
3. **Refactor** — als de tz-conversie in twee plekken duikt (selector + display), extract een `localTimeInTz(utc, tz)` helper.

## Tech-hints

- Expo Push Service: backend stuurt POST naar `https://exp.host/--/api/v2/push/send` met Expo-token + payload. Geen eigen APNs/FCM-setup tot store-submission.
- Tz-opslag: per user 1x bij token-register sturen vanaf device (`Intl.DateTimeFormat().resolvedOptions().timeZone`). Bij wijziging: nieuwe register-call.
- Scheduler-trigger: Railway cron-add-on of een `setInterval` in de Express-server (minder robuust bij restarts; cron is beter). Run elke minuut, idempotent (gebruik een `last_notified_at` per user om dubbele te voorkomen binnen hetzelfde uur).
- HITL-deel: Apple Push Certificate via Apple Developer Console — agent maakt checklist, user genereert .p12, EAS pakt het op tijdens build. Geen werk in deze ticket buiten de checklist; integratie wordt in 3.2 (store-builds) geactiveerd.
- Valkuil: Expo Push is rate-limited (~600/sec per project). Voor v4.0-scale geen probleem; documenteer in code-comment als reminder.
